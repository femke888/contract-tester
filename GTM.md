# Go-to-Market Plan (Local API Contract Tester)

## 1) Positioning (one sentence)
Local-first API contract validation that catches drift from real traffic, without platform lock-in.

## 2) Target users
- API/platform engineers
- QA automation engineers
- Small teams maintaining public/internal APIs

## 3) Core message
- “Catch contract drift from real traffic in minutes.”
- “Offline, CI-friendly, and fast.”
- “Clear error groups with fix hints.”

## 4) 30-day launch checklist
### Week 1 — Setup
- Create a landing page (1 page).
- Create a short demo video (60–90 seconds).
- Publish README badges + `PRICING.md` link.
- Prepare 3 short demos (HAR, curl log, normalized JSON).

### Week 2 — Content + outreach
- Post 3 clips on social (CLI output, HTML report, diff).
- Write 1 blog post: “Detecting API drift from traffic”.
- Share in 2 communities (dev tools, API testing, QA).

### Week 3 — Direct outreach
- Email 20 teams (small SaaS, API-heavy startups).
- Offer a 14-day trial key.
- Ask for 15-minute feedback calls.

### Week 4 — Close + iterate
- Convert 2–5 teams to paid.
- Improve the top 3 pain points from feedback.

## 5) Launch channels
- GitHub release page
- Hacker News “Show HN”
- Reddit (r/devops, r/QualityAssurance, r/API)
- LinkedIn/Twitter

## 6) Pricing
- Standard pricing: `PRICING.md`
- Offer Early Access pricing for first 10–20 customers if needed.

## 7) Metrics to track
- Landing page visits
- Trial keys issued
- Trials → paid conversion
- Weekly active users

## 8) CTA copy
- “Try it on real traffic in 5 minutes.”
- “Download and validate your API now.”
